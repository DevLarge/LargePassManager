Large Pass Refactor Commit-By-Commit Plan

Purpose:
Move from static-heavy architecture to instance-based services while staying runnable after every commit.

Branch:
- git checkout -b refactor/instance-services
- git push -u origin refactor/instance-services
- git push -u secondary refactor/instance-services

Rules for every step:
- Build and run after each commit.
- Keep each commit focused on one change.
- Do not mix UI redesign with architecture refactor in the same commit.

Step 1 - Add application paths config object
Goal:
Create one instance object that owns app directory and file paths.

Files to add:
- src/main/java/largepass/config/AppPaths.java

AppPaths responsibilities:
- Store app directory path.
- Provide methods for vault file path, master key path, logger path.
- Optionally expose helper to resolve child files.

Suggested commit message:
- feat(config): add AppPaths for vault/master/logger locations

Validation:
- mvn test
- mvn javafx:run

Step 2 - Introduce FileService (instance-based)
Goal:
Create a non-static service for file creation/read/write and first-time checks.

Files to add:
- src/main/java/largepass/services/FileService.java

Files to edit:
- none required yet, service can be added without wiring

FileService responsibilities:
- ensureFilesExist()
- isFirstTime()
- readMasterPassword()
- writeMasterPassword(String)
- writeLogger(String)
- deleteSensitiveFiles()

Design notes:
- FileService gets AppPaths in constructor.
- No static mutable fields inside FileService.

Suggested commit message:
- feat(service): add instance-based FileService using AppPaths

Validation:
- mvn test
- mvn javafx:run

Step 3 - Create AppContext
Goal:
Create one object that constructs and stores service instances.

Files to add:
- src/main/java/largepass/config/AppContext.java

AppContext responsibilities:
- Build AppPaths
- Build FileService
- Build CryptoService wrapper (or current EncryptionHandler adapter)
- Expose getters for controllers

Suggested commit message:
- feat(config): add AppContext for service lifecycle

Validation:
- mvn test
- mvn javafx:run

Step 4 - Wire FXMLLoader controller factory in application startup
Goal:
Allow constructor injection into controllers.

Files to edit:
- src/main/java/largepass/LargePassApp.java

Changes:
- Create AppContext in start(...)
- Use FXMLLoader instance (not static load)
- Set controllerFactory to create HubController(context)
- Keep app launching with same Hub.fxml

Suggested commit message:
- refactor(app): use FXMLLoader controller factory for injection

Validation:
- mvn test
- mvn javafx:run
- Confirm Hub scene loads

Step 5 - Convert HubController to constructor injection
Goal:
Stop using static FileHandler calls in HubController.

Files to edit:
- src/main/java/largepass/controllers/HubController.java

Changes:
- Add constructor HubController(AppContext context)
- Store FileService dependency as field
- Replace FileHandler.* with fileService methods
- Keep @FXML fields/methods unchanged for FXML

Note:
- initialize() still works with constructor injection.

Suggested commit message:
- refactor(controller): inject FileService into HubController

Validation:
- mvn test
- mvn javafx:run
- Test first-time flow and returning-user flow

Step 6 - Introduce CryptoService wrapper
Goal:
Hide static EncryptionHandler behind instance API.

Files to add:
- src/main/java/largepass/services/CryptoService.java

Files to edit:
- src/main/java/largepass/config/AppContext.java
- src/main/java/largepass/controllers/HubController.java

Changes:
- CryptoService offers encrypt/decrypt instance methods
- HubController uses CryptoService instead of static EncryptionHandler directly

Suggested commit message:
- refactor(service): add CryptoService wrapper and inject into HubController

Validation:
- mvn test
- mvn javafx:run

Step 10 - Final cleanup commit
Goal:
Remove dead code and tighten naming/documentation.

Files to edit:
- any obsolete classes/imports/usages

Changes:
- Remove unused static helpers
- Remove stale imports
- Keep final architecture consistent

Suggested commit message:

Validation:
- mvn test
- mvn javafx:run

Merge checklist:
- All tests pass.
- App launches and can login.
- First-time setup still works.
- Returning-user validation still works.
- Wrong-password behavior still works.
- No static mutable file state in runtime path.

Merge commands:
- git checkout main
- git pull
- git merge --no-ff refactor/instance-services
- git push origin main
- git push secondary main

Optional squash strategy:
If you want cleaner history, use interactive rebase before merge:
- git checkout refactor/instance-services
- git rebase -i main

Recovery tip:
If one step breaks, do not continue.
- git status
- git add only stable files
- git commit or git stash
- fix that step first, then proceed
